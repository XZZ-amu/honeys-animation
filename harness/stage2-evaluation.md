# 阶段 2：评估机制

动画生成后，按以下三层评估。每层有不同的判断主体和处理方式。

---

## 三层评估

### 第一层：及格线（系统自动）

生成后自动检查，不通过 = 直接打回重做，不给用户看。

检查项：
- [ ] 代码能跑（浏览器打开不报错、不白屏）
- [ ] 有动画效果（不是静态页面）
- [ ] 循环播放正常（如果 spec 要求循环）
- [ ] spec 中的硬性要素存在（spec 说有棒就得有棒、说有伞盖就得有伞盖）
- [ ] 颜色使用了 spec 指定的方案（不是随机的）
- [ ] 元素类型正确（spec 说字母就是字母，说形状就是形状）

### 第二层：品质线（系统辅助标注）

通过及格线后，系统辅助标注可能有问题的地方，然后给用户看。

标注项：
- 物理行为是否像真实世界（对照 rules/01 的现实锚定规则）
- 跟参考图/草图的结构是否一致
- 配色比例是否符合 style-tokens.css 的规则
- 是否有"改 A 坏 B"的迹象（对比 spec 中锁定的字段）

### 第三层：审美线（用户判断）

系统无法自动评估，必须由用户看：
- 整体感觉对不对
- 节奏/韵律舒不舒服
- 细节品质满不满意

## 批量生成触发条件

不是默认行为。只在以下情况触发（作为意图澄清的兜底）：
- 迭代 2-3 轮用户还在说"不对"
- 用户描述越来越模糊（"就是那种感觉嘛"）
- 选项确认时用户选了"都不是"

触发时：出 3 个不同方向的版本让用户选，帮助缩小意图范围。
