# 规则 06：改细节就改细节，不要重做整个动画

## 什么错
为了把"字母淡出消失"换成"沙粒碎裂消失"，重写了整个动画（v19）。新版的布局、时间线、风的节奏全变了，之前调好的东西全丢了。

## 为什么错
把"替换一个模块"当成了"重新实现整个系统"。消失方式只是飞行阶段的最后一个环节，跟布局、时间线、风的节奏没有关系。应该只改那一个环节。

## 怎么防

**判断标准**：这次修改需要动几个模块？

- 只动 1 个模块 → 在现有代码上修改那个模块，其余不动
- 需要动 2-3 个模块 → 逐个修改，每改一个验证一次
- 底层架构要变（比如渲染方式从 fillText 变成像素点）→ 可以新建文件，但必须把不需要改的代码原封搬过来，不重写

**执行规则**：
- 新建文件时，先复制原文件，再在副本上改
- 给执行 agent 的 prompt 里明确列出"以下必须保持不变"的清单
- 改完后对比两个版本，确认只有目标模块变了
