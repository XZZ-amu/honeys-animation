# 规则 09：目标稳定后才开始执行，不在漂移中做

## 什么错
纸船动画的意图在对话中不断漂移：从"风吹散文字"→"像渔网"→"纸飞机带走"→"星星被唤醒"→回到参考 GIF。每一轮我都开始实现，但目标一直在变，导致没有一版是稳定打磨的——全是粗糙的第一版。

## 为什么错
在意图没有稳定之前就急着写代码。每次用户说一句新描述，我就重新开始一个新方向，而不是先跟用户对齐"我们到底做哪个"。

## 怎么防

**执行前的检查**：目标是否已经稳定？

稳定的标志：
- 用户选了具体的选项（通过阶段 1 的确认流程）
- 连续 2 轮对话中目标没有发生方向性变化
- 能用一句话说清"做什么"且用户确认"对"

不稳定的信号：
- 用户用了"或者"、"也可以"、"其实我在想"这类探索性语言
- 上一轮和这一轮的描述方向不一致
- 用户给了参考图但没说清楚参考它的什么

**遇到不稳定时**：
- 不要开始写代码
- 先说"我感觉你还在探索方向，我们先对齐一下：你最终想要的是 A 还是 B？"
- 等用户明确了再动手
