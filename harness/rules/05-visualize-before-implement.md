# 规则 05：写代码前必须先完成视觉想象

## 什么错
风吹散文字的动画，第一版字母按位置顺序逐个消失，像在按删除键，不像真实的风吹。

## 为什么错
从需求直接跳到了"查参数表 → 拼代码"，跳过了"这个画面在真实世界中到底长什么样"的想象步骤。参数表能给你正确的物理数值，但不能告诉你"哪些字母先飞、飘走的姿态是什么、节奏有没有呼吸感"这些让画面自然的细节。

## 怎么防

**强制检查点**：在写任何动画代码之前，必须先用文字回答以下问题（写在 prompt 里给执行 agent）：

1. **时间零点**：动画开始前，画面是什么状态？完全静止？还是有微小的活态？
2. **触发瞬间**：变化开始的第一个 0.5 秒，观众应该看到什么？（预兆/前奏）
3. **过程中的不均匀性**：哪些元素先动、哪些后动？为什么？（不能是"按位置顺序"）
4. **运动的姿态**：元素在运动过程中像什么？（被拖拽？弹飞？飘荡？坠落？）
5. **节奏的呼吸感**：是匀速的吗？如果不是，哪里快哪里慢？有没有一阵一阵的感觉？
6. **结束状态**：最终画面是什么？怎么过渡到结束？

**错误模式识别**：如果 prompt 里只有参数值（速度=XX、重力=YY、角度=ZZ）而没有上面这些"画面描述"，说明跳过了想象步骤，必须退回来补。

## 正确做法

```
需求 → 在脑中想象真实画面 → 用文字描述画面细节（回答上面6个问题） → 从描述反推参数 → 写代码
```

不是：
```
需求 → 查参数表 → 拼代码
```
