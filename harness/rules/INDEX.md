# 规则索引

执行前快速扫一遍。触发时再看对应文件的详情。

1. **物理锚定现实** — 物理参数从真实世界推导，不从情绪推导
2. **不跳过阶段 1** — 再明确的需求也要走意图确认，至少确认结构形态
3. **选项覆盖结构** — 选项区分"长什么样"，不是"怎么动"
4. **复用带上下文** — 搬代码时连同布局参数一起搬，布局优先物理适配
5. **先想象再实现** — 写代码前先用文字描述完整的视觉画面（6 个问题）
6. **改细节不重做** — 只需要改一个模块时，不要重写整个动画
7. **心智模型优先** — 先选对"这个效果的本质是什么"，再动手
8. **超出能力就说** — 把握低于 50% 时直接告知，给替代方案
9. **目标稳定才动手** — 意图还在漂移时不写代码，先对齐方向
10. **具体约束覆盖软约束** — 参考图（具体）优先于文字描述（模糊），冲突时以参考图为准并跟用户确认
