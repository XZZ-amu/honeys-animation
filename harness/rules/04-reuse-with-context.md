# 规则 04：复用代码必须连同上下文一起搬

## 什么错
把 V4 的物理引擎代码搬到新动画中，但没有搬配套的布局参数。结果铃铛跑到伞盖上面去了。

## 为什么错
代码模块之间有耦合关系。物理引擎的 segLen、gravity 决定了链条垂多长；布局决定了"可以垂多长的空间"。两者必须匹配。只搬一个不搬另一个 = 接口对不上。

## 怎么防

**强制检查点**：当复用已有动画的代码模块时，必须回答：

1. 这个模块依赖哪些外部参数？（如：segLen 依赖可用空间高度）
2. 新动画的这些参数跟原来一样吗？
3. 如果不一样，哪些参数需要反算适配？

**正确做法**：布局优先，物理适配布局。
1. 先确定布局（东西在哪、占多大空间）
2. 计算物理参数能用的空间（如 chain 区域 = 伞盖底部到画面 75%）
3. 反算 segLen 等参数适配这个空间
4. 其他物理参数（gravity、damping、wind）可以直接复用

**绝不能**：直接把物理代码贴过来，让执行 agent"自己看着办"布局。
