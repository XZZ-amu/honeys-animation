# 阶段 5：持续改进

让系统从每次创作中学习，越跑越好。

---

## 记录结构

每次创作完成后，记录以下信息：

### 创作日志（存 harness/logs/）

文件命名：`{日期}-{动画编号}-{主题}.md`

```markdown
# 创作记录

## 基本信息
- 动画文件：animations/XX-xxx.html
- 用户原始输入：（原话）
- 最终 spec：（填好的模板）
- 迭代轮次：N 轮
- 最终状态：满意 / 部分满意 / 放弃

## 意图澄清过程
- 选项确认：用户选了什么
- 情绪确认：用户选了什么
- 参考图：有/无，提取了什么

## 迭代历史
- 第 1 轮：改了什么，用户反馈什么
- 第 2 轮：...

## 成功点（做对了什么）
- ...

## 失败点（做错了什么，原因）
- ...

## 是否产生新规则
- 是 → 规则编号和内容
- 否
```

### 规则（存 harness/rules/）

从失败中固化的约束。格式：什么错 → 为什么错 → 怎么防。
发现新错误模式后立即写入，下次执行前强制检查。

### 成功模式（存 harness/patterns/）

从成功中提取的可复用经验。格式：

```markdown
# 模式名称

## 适用场景
什么类型的动画适用这个模式

## 推荐参数范围
- 物理参数：...
- 视觉参数：...

## 验证过的案例
- animations/XX：这组参数效果好
- animations/YY：同样适用

## 注意事项
- 什么情况下不适用
```

### 用户偏好（存 harness/preferences/）

从用户的选择和反馈中积累的审美偏好：

```markdown
# 用户偏好

## 已确认的偏好
- 喜欢干净锐利的线条，不喜欢 glow/blur
- 物理行为要真实，不接受夸张
- 微风 > 大风，细腻 > 张扬
- ...

## 偏好来源
- 偏好 1：来自 XX 动画的反馈"不够真实"
- 偏好 2：来自 YY 动画的确认"这个风感觉对了"
```

## 改进循环

```
每次创作结束
  │
  ├─ 用户满意
  │   ├─ 记录成功点
  │   ├─ 提取可复用的参数组合 → harness/patterns/
  │   └─ 提取用户偏好 → harness/preferences/
  │
  └─ 用户不满意
      ├─ 记录失败点和原因
      ├─ 是否是新的错误模式？
      │   ├─ 是 → 写入 harness/rules/
      │   └─ 否 → 检查已有规则为什么没防住
      └─ 更新相关模式的"注意事项"
```

## 执行时机

- **创作前**：加载 rules/ + patterns/ + preferences/，注入到执行 agent 的 context
- **创作中**：实时检查是否违反 rules/
- **创作后**：填写创作日志，判断是否需要新增 rules/patterns/preferences
