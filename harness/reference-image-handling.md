# 参考图输入处理规则

当用户提供参考图（精美图片、GIF 截帧、手绘草图）时，系统的处理流程。

---

## 第一步：判断输入类型和意图

### 输入类型识别

| 类型 | 特征 | 系统行为 |
|------|------|---------|
| 精美参考图 | 完整的设计作品、截图、照片 | 提取视觉风格 + 结构 |
| GIF/视频截帧 | 动态内容的一帧 | 提取运动暗示 + 视觉风格 |
| 手绘草图 | 粗糙线条、示意性的 | 只提取结构和布局，不提取风格 |

### 意图判断

根据用户附带的话判断：

| 用户说了什么 | 意图类型 | 系统提取什么 |
|-------------|---------|-------------|
| "做一个跟这个一样的，但把 XX 换成 YY" | 复刻结构 + 换元素 | 完整结构 + 布局 + 风格，替换指定元素 |
| "我喜欢这个配色" / "用这个颜色" | 只取配色 | 提取色板（主色/辅色/点缀色及比例） |
| "参考它这个 XX 效果" / "像这个 XX 的感觉" | 取局部特征 | 只提取用户指定的那个维度 |
| "造型跟这张走" / "结构参考这个" | 取结构/造型 | 提取结构关系和造型语言 |
| 什么都没说，只丢了一张图 | 不确定 | 反问："你想参考它的什么？A.整体复刻 B.取配色 C.参考某个局部" |

---

## 第二步：从图中提取信息

### 精美参考图的提取清单

**可以提取的**：
- 结构关系（什么在什么上面、怎么连接、几个元素）
- 造型语言（圆润/锐利/几何/有机/手绘感）
- 色板（列出具体色值、判断主色/辅色/点缀色比例）
- 布局比例（元素占画面多大、留白多少、重心在哪）
- 元素类型（用了什么形状、什么图案）

**不能照抄的**：
- 物理行为参数（图片里元素飞了多远 ≠ 动画里应该飞多远）
- 精确像素尺寸（参考图的分辨率和我们的画布不一样）

### 手绘草图的提取清单

**只提取**：
- 结构（什么连着什么、层级关系）
- 布局（大概位置和比例）
- 元素类型和数量

**不提取**：
- 线条风格（草图的粗糙不是目标风格）
- 精确比例（随手画的不精确）
- 颜色（除非明确上了色且用户指出"用这些颜色"）

### GIF/视频截帧的提取清单

**可以提取**：
- 运动方向和方式的暗示（从模糊/拖影判断运动感）
- 视觉风格
- 结构布局

**不能照抄**：
- 物理参数（仍然从现实世界推导）
- 帧率/速度（截帧不反映真实速度）

---

## 第三步：提取结果填入 spec

将提取的信息填入 animation-spec-template.md 的对应字段：

- 结构 → 视觉描述
- 色板 → 配色方案
- 造型语言 → 补充到硬性约束
- 运动暗示 → 动画类型（但物理参数仍从现实推导）

---

## 第四步：确认提取是否正确

提取完成后，向用户确认："我从这张图里看到了 XX，准备按 YY 方式用在动画里，对吗？"

只有用户确认后才进入执行。如果用户说"不是"，请用户补充说明。

---

## 核心原则

1. **参考图是"感觉参照"，不是"施工图纸"**——提取的是视觉意图，不是精确参数
2. **物理行为永远听现实世界，不听参考图的静态布局**——图里元素飞了多远不代表动画里该飞多远
3. **草图只提供结构，不提供风格**——风格从 token 系统或情绪选择来
4. **不确定就问**——如果无法判断用户想参考什么，反问比猜测好
