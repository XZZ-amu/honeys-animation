# 情绪 × 动作类型 → 运动方式映射表

系统根据用户选的"情绪"和"动作类型"自动选择合适的运动方式组合。用户不需要知道这些技术名词。

基于常识定义的 V1 版本，后续通过案例迭代修正。

---

## 入场方式（元素第一次出现时）

| 情绪 | 推荐入场 | 参数倾向 |
|------|---------|---------|
| 调皮捣蛋 | overshoot（弹性过冲） | amp 大、freq 高、decay 中 — 蹦出来的感觉 |
| 温暖浪漫 | gentle ease-out | 慢慢显现，柔和停下 |
| 活泼热闹 | bounce（弹跳） | 弹进来，有节奏感 |
| 神秘魔法 | fade + slow ease-out | 从无到有，缓慢浮现 |
| 轻柔飘散 | gentle ease-out + oscillation | 轻轻飘入，到位后微微浮动 |
| 干净利落 | sharp ease-out | 快速到位，干脆停下，无多余动作 |

---

## 退场方式（元素消失时）

| 情绪 | 推荐退场 | 参数倾向 |
|------|---------|---------|
| 调皮捣蛋 | bounce out（弹走） | 跳着离开 |
| 温暖浪漫 | fade + gentle ease-in | 慢慢变淡飘走 |
| 活泼热闹 | sharp ease-in（快速飞出） | 嗖一下就没了 |
| 神秘魔法 | fade + scale down | 缩小消失，像魔法收回去 |
| 轻柔飘散 | inertia + sand-erosion | 被风带走 + 逐渐碎裂 |
| 干净利落 | sharp ease-in | 加速离开，不回头 |

---

## 持续运动（元素停留时的活态）

| 情绪 | 推荐持续运动 | 参数倾向 |
|------|------------|---------|
| 调皮捣蛋 | wiggle + 不规则节奏 | 坐不住，随机抖动，幅度小但频率不固定 |
| 温暖浪漫 | oscillation（慢速呼吸） | 上下微微浮动，freq=0.3-0.5Hz |
| 活泼热闹 | oscillation（中速） + 旋转 | 有节奏地跳动，freq=1-2Hz |
| 神秘魔法 | oscillation（极慢） + fade pulse | 幽幽飘浮，透明度微微变化 |
| 轻柔飘散 | sway（横向摆动） | 像风中的叶子，左右轻摇 |
| 干净利落 | 无 / 极微小呼吸 | 几乎静止，最多 1-2px 微动 |

---

## 特殊动作类型的运动方式

| 动作 | 推荐运动方式 | 为什么 |
|------|------------|--------|
| 弹跳/跳出来 | bounce | 有重力、有反弹、有衰减 — 像真实的跳 |
| 飘走/吹走 | inertia + sway | 被持续外力带走 + 在风中摆动 |
| 碎裂/消散 | sand-erosion | 从迎风侧逐粒脱落 |
| 旋转飞入 | path follow + rotation | 沿弧线进入 + 自旋 |
| 坠落 | ease-in（加速） + bounce | 先加速落下，着地反弹 |
| 浮现 | scale up + overshoot | 从 0 放大到 100%，带轻微过冲 |
| 抖动后恢复 | wiggle decay | 高频震动逐渐归零 |
| 波浪/级联 | stagger + 任意运动 | 同一运动加时间差 |

---

## 使用规则

1. 系统根据 spec 里的"情绪"和"动画描述"自动选择，填入执行 agent 的 prompt
2. 用户不需要知道这些术语——系统自动翻译
3. 如果选的运动方式不对，进入迭代修改流程（阶段 3）
4. 每次确认"这个运动方式对了"时，记录到 patterns/ 中积累
