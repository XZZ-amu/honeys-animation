# 阶段 1：意图澄清流程

当收到用户的动画创作需求时，系统必须按以下流程执行。不允许跳过。

---

## 流程

```
用户输入（文字/草图/参考图）
  │
  ├─ 判断输入类型
  │   ├─ 纯文字 → 进入选项确认
  │   ├─ 草图 → 提取结构意图，不提取风格
  │   └─ 参考图 → 判断用户想参考什么（见下方）
  │
  ├─ 选项确认（最多 1-2 题）
  │   ├─ 第一题必须区分"结构/形态"（长什么样），不是"运动方式"
  │   ├─ 用户选一个，或说"都不是"
  │   └─ 如果都不是 → 请用户补充描述或给草图
  │
  ├─ 情绪确认（1 题）
  │   ├─ 选项之间的区别是情绪/感受，不是技术参数
  │   └─ 用户选一个，或自己说一个感觉
  │
  └─ 产出填好的 spec（见 animation-spec-template.md）
      → 交给执行 agent
```

## 参考图处理规则

用户给参考图时，根据附带的话判断意图：
- "做一个跟这个一样的，但改 XX" → 复刻结构 + 换指定元素
- "我喜欢这个配色" → 只提取色板
- "参考它这个 XX 效果" → 只提取指定的局部特征
- 什么都没说 → 反问"你想参考它的什么？"

**从参考图提取的原则**：
- 视觉风格（配色、造型语言）→ 复刻
- 物理行为（飞多远、多快）→ 不照抄图片布局，遵循现实世界

## 草图处理规则

从草图中只提取：
- 结构（什么在什么上面、怎么连接）
- 布局（大概的位置和比例关系）
- 元素类型（什么形状、多少个）

不从草图提取：
- 线条风格（草图的粗糙线条不是用户要的最终风格）
- 精确尺寸（草图比例是随手画的，不是精确的）
- 颜色（除非草图明确上了色）

## 选择题设计规则

- 总数不超过 2 题
- 选项 2-3 个 + "自己说一个"逃生口
- 第一题区分结构形态
- 第二题区分情绪/感受
- 不用选择题收集参数（参数由系统推导）

## 参数推导规则

用户不操心参数，系统自动推导：
- 物理参数（速度、寿命、幅度、重力）→ 从"这是什么物体"的现实行为推导
- 视觉参数（颜色、发光、材质）→ 从"用户选的情绪"推导
- 两者独立，不互相污染
