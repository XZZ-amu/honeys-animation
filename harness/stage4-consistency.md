# 阶段 4：一致性维护

防止系列作品之间的视觉漂移。所有动画生成时必须遵守此流程。

---

## 三层防护

### 第一层：Token 文件（硬约束）

文件：`harness/style-tokens.css`

所有动画必须引用这些变量。生成代码时不允许硬编码色值、字号、线宽——必须使用 token 变量或 token 定义的值。

### 第二层：风格规范（软约束）

文件：`harness/style-spec.md`

执行 agent 生成前必须阅读。描述了"什么场景用什么颜色"、"禁止什么"、"造型语言是什么"。

### 第三层：后置验证（兜底）

生成后检查是否违反规则：
- [ ] 使用的颜色是否全部在 token 色板内
- [ ] 线条粗细是否使用了 token 定义的三个等级
- [ ] 背景色是否为 token 定义的背景色
- [ ] 字体是否为无衬线粗体
- [ ] 有无 glow/shadow/blur（禁止）
- [ ] 动效缓动是否使用了 token 定义的 ease 函数

## 复用代码时的一致性规则

当复用已有动画的代码模块时（参考 rules/04）：

1. 确认模块依赖哪些外部参数
2. 新动画的这些参数是否兼容
3. 布局优先：先定布局，再让物理参数适配
4. 不允许直接贴代码然后"看着办"

## 跨动画的一致性

同一系列的多个动画之间：
- 色板不变（每个动画可以选不同主色，但色板本身一致）
- 造型语言一致（都圆润或都锐利，不混搭）
- 动效节奏一致（同样的 ease 函数、同样的时长体系）
- 背景一致

## 当需要新风格时

如果新动画明确需要不同的风格（如草图风铃的"柔和梦幻"vs 仙女棒的"干净锐利"），说明这是一个**新系列**，需要：
1. 创建新的 token 文件（如 `style-tokens-dreamy.css`）
2. 在 spec 中声明使用哪套 token
3. 同一系列内保持一致，不同系列之间可以不同
