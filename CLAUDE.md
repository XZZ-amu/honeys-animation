# Honeys 动画设计

用 AI 生成字符动画（HTML），通过 5 阶段 harness 设计练习，学会如何让 agent 可靠地完成创作任务。

## 项目目的

这不是一个"做动画"的项目，而是一个**练习 harness 设计**的项目。动画是载体，核心是学会：
- 怎么让 agent 理解创作意图
- 怎么评估创作结果的好坏
- 失败时怎么诊断和修复
- 长时间迭代中怎么管理 context
- 怎么从历史数据中改进系统

## 用户画像

- 产品设计师，不写代码
- 用 Claude Code 指挥 agent 工作
- 目标：掌握 harness 设计的 5 个阶段，能独立设计让 agent 可靠工作的系统

## Harness 5 阶段框架

### 阶段 1：任务理解 + 工具就绪
> 让 agent 知道做什么、有什么可用

练习重点：
- 怎么描述一个动画的创作意图（不是写 spec，是让 agent "懂"你要什么）
- 工具边界：agent 能用 HTML/CSS/JS，输出是单个 .html 文件
- 约束声明：时长、帧率、风格、字符集等

### 阶段 2：执行 + 分离评估
> 做事的和判断的必须是不同角色

练习重点：
- 创作 agent 和评估 agent 分离
- 评估标准要具体可打分（不是"好不好看"，而是"动画是否流畅/是否表达了主题/是否在约束内"）
- 设计评分维度和评分规则

### 阶段 3：失败修复策略
> 不是重试，而是问"缺了什么能力"

练习重点：
- 动画不达标时，诊断是哪个环节出了问题
- 区分：意图理解错了 vs 技术实现错了 vs 审美判断错了
- 针对性补充信息，而不是"再来一次"
- 卡住时强制换方向（比如换动画风格、降低复杂度）

### 阶段 4：Context 生命周期管理
> 长时间运行中防止信息腐烂

练习重点：
- 多轮迭代后，哪些 context 要保留、哪些要丢弃
- 什么时候该重启一个新 session
- 怎么把"已验证的偏好"沉淀下来，不让每次都重新对齐

### 阶段 5：监控 + 持续改进
> 追踪历史错误模式，反哺 harness 设计

练习重点：
- 记录每次创作的评分和失败原因
- 发现模式：哪类动画容易失败？什么描述方式成功率高？
- 用发现改进 prompt/评估标准/任务描述模板

## 输出规范

- 每个动画 = 一个独立的 .html 文件，放在 `animations/` 目录
- 可直接用浏览器打开查看效果
- 文件命名：`{序号}-{主题简述}.html`，如 `01-hello-world.html`

## 练习节奏

每次练习聚焦一个阶段。不要同时练所有阶段。
从阶段 1 开始，做 2-3 个动画把阶段 1 练熟，再进入阶段 2。
