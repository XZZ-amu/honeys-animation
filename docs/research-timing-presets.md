# 动效时长与缓动曲线调研

> 调研目标：找到专业动效软件和设计规范中"开箱即好"的默认时长和缓动曲线参数，作为我们字符动画系统的预设默认值。

---

## 核心发现

**为什么 C4D/AE 做出来的动画"开箱就快"？**

1. **时长偏短**：专业软件的默认 transition 普遍在 0.2-0.5s，而非新手直觉中的 1-2s
2. **缓动不对称**：退场比入场快 20-40%，这符合人类注意力规律（消失不需要细看）
3. **不用线性**：任何运动都有加速/减速，线性运动是"最不自然"的
4. **曲线有"甩尾"**：好的退场曲线是先慢后快（ease-in），给人"被弹走"的感觉

## 一、After Effects 的动效参数

### 1.1 内置 Easy Ease 系列

AE 的 keyframe easing 是基于速度图（Speed Graph），对应的贝塞尔参数：

| 缓动类型 | 贝塞尔参数 cubic-bezier(x1,y1,x2,y2) | 特征 |
|----------|--------------------------------------|------|
| Easy Ease (默认F9) | (0.33, 0, 0.67, 1) | 两头慢中间快，万能曲线 |
| Easy Ease In | (0.42, 0, 1, 1) | 开头慢，结尾匀速冲出 |
| Easy Ease Out | (0, 0, 0.58, 1) | 开头匀速，结尾减速停稳 |
| 强力 Ease In（手动调75%影响） | (0.55, 0.085, 0.68, 0.53) | 更明显的慢启动 |
| 强力 Ease Out（手动调75%影响） | (0.25, 0.46, 0.45, 0.94) | 更明显的减速 |

### 1.2 内置 Transition 预设时长

AE 默认 composition 30fps，内置预设的默认时长：

| 预设类型 | 默认帧数 | 换算时长 | 备注 |
|----------|----------|----------|------|
| Fade In/Out | 15帧 | 0.5s | 最基本的淡入淡出 |
| Scale Up/Down | 15帧 | 0.5s | 缩放进出 |
| Fly In/Out | 10-15帧 | 0.33-0.5s | 飞入飞出画面 |
| Slide (滑动) | 10帧 | 0.33s | 小距离位移 |
| Wipe | 20帧 | 0.67s | 擦除转场 |
| 文字 Animate In (逐字) | 每字符 2-3帧 | 每字符 0.067-0.1s | 20个字符总计 1.3-2s |

### 1.3 MoGraph 文字动画（AE Text Animator）

AE 文字动画器的默认行为：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 单字符动画时长 | 4-6 帧 (0.13-0.2s) | 一个字从"开始动"到"到位" |
| 字符间延迟（Offset） | 2-4 帧 (0.067-0.13s) | 相邻字符的启动间隔 |

| Smoothness | 30-50% | 控制相邻字符的动画重叠程度 |
| 常用 Range Selector 形状 | Ramp Up / Triangle | 控制"波浪式"扩散效果 |

| 参数 | 默认/典型值 | 说明 |

| 曲线类型 | 行为 | 对应贝塞尔 |

| 阶段 | 时长 | 曲线 | 说明 |
|------|------|------|------|

| 加速飞出 | 8-12帧 (0.27-0.4s) | Ease In (越来越快) | 字符加速离开 |

## 四、Apple Human Interface Guidelines

### 4.1 iOS/macOS 动画时长规范

| 动画类型 | 推荐时长 | 说明 |
|----------|----------|------|
| 轻触反馈 | 0.1s | 按钮高亮等 |
| 小元素出现/消失 | 0.2-0.25s | 按钮、图标、Toast |
| 中等元素转场 | 0.3-0.35s | 卡片、模态弹窗 |
| 全屏转场 | 0.35-0.5s | 页面切换、导航 |
| 大面积/复杂转场 | 0.5-0.7s | 展开/折叠整个界面 |

### 4.2 Apple 的核心原则

- **退场比入场快 30%**：入场 0.35s → 退场 0.25s
- **大面积比小面积慢**：距离越远需要时间越长，但不是线性比例
- **Spring 优先**：iOS 从 iOS 7 开始大量使用弹性动画

### 4.3 iOS 默认 Spring 参数

| 用途 | Spring 参数 | 等效时长 | 说明 |
|------|-------------|----------|------|
| 系统默认 (UIView.animate) | damping=1.0, response=0.35s | 0.35s | 无回弹 |
| 轻微弹性 | damping=0.7, response=0.5s | 0.6s | 轻微过冲 |
| 明显弹性 (app icon drop) | damping=0.5, response=0.6s | 0.8s | 明显弹跳 |
| 强弹性 (bouncy) | damping=0.3, response=0.8s | 1.2s+ | 多次弹跳 |

### 4.4 Apple 缓动曲线（非 Spring 场景）

| 用途 | cubic-bezier | 说明 |
|------|-------------|------|
| 默认(EaseInOut) | (0.25, 0.1, 0.25, 1) | 系统标准 |
| 入场(EaseOut) | (0, 0, 0.2, 1) | 快来慢停 |
| 退场(EaseIn) | (0.4, 0, 1, 1) | 慢启快走 |

## 五、Material Design 动效规范

### 5.1 Duration Tokens (MD3)

| Token | 时长 | 适用场景 |
|-------|------|----------|
| duration-short1 | 50ms | 极简反馈（涟漪起始） |
| duration-short2 | 100ms | 小元素淡入淡出 |
| duration-short3 | 150ms | 小范围位移 |
| duration-short4 | 200ms | 按钮、小卡片出现 |
| duration-medium1 | 250ms | 标准入场 |
| duration-medium2 | 300ms | 标准退场、中等元素 |
| duration-medium3 | 350ms | 较大元素 |
| duration-medium4 | 400ms | 大面积展开 |
| duration-long1 | 450ms | 页面转场 |
| duration-long2 | 500ms | 复杂转场 |
| duration-long3 | 550ms | — |
| duration-long4 | 600ms | — |
| duration-extra-long1 | 700ms | 超大面积动画 |
| duration-extra-long2 | 800ms | — |
| duration-extra-long3 | 900ms | — |
| duration-extra-long4 | 1000ms | 极少使用 |

### 5.2 Easing Tokens (MD3)

| Token | cubic-bezier | 用途 | 说明 |
|-------|-------------|------|------|
| easing-standard | (0.2, 0, 0, 1) | 通用动画 | 快起慢停，最常用 |
| easing-standard-decelerate | (0, 0, 0, 1) | 入场 | 极快起步→缓慢到位 |
| easing-standard-accelerate | (0.3, 0, 1, 1) | 退场 | 缓慢启动→快速离开 |
| easing-emphasized | (0.2, 0, 0, 1) | 强调动画 | 同 standard，配更长时长 |
| easing-emphasized-decelerate | (0.05, 0.7, 0.1, 1) | 强调入场 | 弹性感减速 |
| easing-emphasized-accelerate | (0.3, 0, 0.8, 0.15) | 强调退场 | 明显加速消失 |

### 5.3 MD 的核心规则

- **退场用 accelerate**：物体加速离开视野
- **入场用 decelerate**：物体减速进入视野
- **退场时长 < 入场时长**：退场不需要用户注意
- **移动距离越远，时长越长**：但用 sqrt 比例而非线性

## 六、Motion Design 行业经验

### 6.1 "舒服的" 时长范围

| 场景 | 最低 | 甜区 | 最高 | 超过则"拖沓" |
|------|------|------|------|-------------|
| 按钮反馈 | 50ms | 100-150ms | 200ms | >300ms |
| 元素出现 | 150ms | 200-350ms | 500ms | >700ms |
| 元素消失 | 100ms | 150-250ms | 400ms | >500ms |
| 页面转场 | 200ms | 300-500ms | 700ms | >1000ms |
| 复杂编排动画 | 500ms | 800-1500ms | 2500ms | >3000ms |
| 字符动画(逐字飞出) | 400ms | 600-1200ms | 2000ms | >2500ms |

### 6.2 情绪与速度对应

| 情绪/风格 | 单元素时长 | 曲线特征 | 适用场景 |
|-----------|-----------|----------|----------|
| 轻快/灵动 | 0.15-0.3s | 强 ease-out，快速到位 | 游戏UI、年轻化产品 |
| 专业/克制 | 0.25-0.4s | 对称 ease-in-out | 商务工具、阅读类 |
| 优雅/从容 | 0.4-0.7s | 柔和曲线，小幅过冲 | 奢侈品、艺术类 |
| 戏剧感/爆发 | 0.1-0.2s(爆发) + 0.5-1s(余韵) | 极强 ease-in 后拖尾 | 游戏特效、电影 |
| 轻柔/梦幻 | 0.8-2s | 几乎线性的缓动 | 背景氛围动画 |
| 科技/精确 | 0.2-0.35s | 锐利 ease-out，无过冲 | 数据可视化 |

### 6.3 Disney 12 原则中的 Timing 数值

Disney Animation 的 timing/spacing 原则映射到数字动效：

| 原则 | 数值建议 | 说明 |
|------|----------|------|
| Slow In / Slow Out | 动画首尾各占总时长 25-35% | 两头慢中间快的分配 |
| Timing (轻物体) | 2-4帧@24fps = 0.08-0.17s | 乒乓球弹跳一次 |
| Timing (重物体) | 8-12帧@24fps = 0.33-0.5s | 保龄球落地一次 |
| Anticipation | 动作前 3-5帧准备动作 = 0.1-0.2s | "蓄力"感 |
| Follow Through | 动作后 4-8帧余韵 = 0.17-0.33s | 惯性拖尾 |
| Overlapping Action | 次要部分延迟 2-4帧 | 头发、衣角比身体晚动 |
| Exaggeration (scale) | 过冲 10-30% | 弹性变形 |

### 6.4 12fps "动画感" 阈值

动画行业共识：
- **低于 8fps**：看起来像幻灯片
- **12fps**：最低限度的"动画感"（传统动画标准）
- **24fps**：电影流畅度
- **30fps**：视频/游戏标准
- **60fps**：高级流畅度

对应到时长：一个动作要"看起来是动画"，至少需要 3-4 帧。在 60fps 下 = 最少 50-67ms。

## 七、综合建议：对我们字符动画系统的推荐默认值

### 7.1 各场景推荐参数

| 场景 | 总时长 | 单字时长 | stagger间隔 | 缓动曲线 | cubic-bezier |
|------|--------|----------|-------------|----------|--------------|
| **元素飞出画面** | 0.4-0.6s | — | — | Ease In (加速离开) | (0.3, 0, 1, 1) |
| **文字消散/飘走** | 0.8-1.5s(整体) | 0.3-0.5s | 0.03-0.06s | Ease In + 物理漂移 | (0.4, 0, 0.8, 0.15) |
| **轻柔飘散(蒲公英)** | 1.5-3s(整体) | 0.8-1.5s | 0.05-0.1s | 近线性+微弱ease | (0.1, 0, 0.3, 1) |
| **快速爆炸/弹飞** | 0.3-0.6s(整体) | 0.15-0.3s | 0-0.02s(几乎同时) | 强 Ease In | (0.5, 0, 1, 0) |
| **淡入** | 0.2-0.4s | — | — | Ease Out | (0, 0, 0.2, 1) |
| **淡出** | 0.15-0.3s | — | — | Ease In | (0.4, 0, 1, 1) |
| **弹性动画(bounce)** | 0.6-1.0s | — | — | Spring | damping=0.4-0.6, freq=10-15Hz |

### 7.2 Stagger（依次触发）的黄金比例

字符动画的"波浪感"核心参数：

| 字符数 | 建议 stagger | 整体耗时(含单字动画) | 来源 |
|--------|-------------|---------------------|------|
| 5-10字 | 40-60ms/字 | 0.5-1.0s | AE text animator 默认 |
| 10-20字 | 30-50ms/字 | 0.8-1.5s | C4D Delay Effector |
| 20-50字 | 20-40ms/字 | 1.0-2.5s | 越多字间隔越短，否则太长 |
| 50+字 | 10-25ms/字 | 1.5-3.0s | 大面积需要更快"扫过" |

**关键洞察**：stagger 不应该线性增长。字越多，间隔应该越短。公式近似：
```
stagger_ms = 60 / sqrt(字符总数 / 10)
```

### 7.3 "C4D 感"的秘密

C4D MoGraph 之所以"开箱即好"，核心不是时长短，而是：

1. **Ease In 退场曲线**：物体加速飞走，不是匀速飘走
2. **Stagger 带随机**：字符不是严格依次飞，而是有 ±20% 的时间抖动
3. **多属性叠加**：位移 + 旋转 + 缩放 + 透明度同时动，但各自延迟 1-3帧
4. **位移距离够大**：飞出画面不是飞 100px，是飞到画面外 150%
5. **overshoot**：入场时有 5-15% 的过冲再回弹

## 八、与现有 physics-presets.json 的对比分析

### 8.1 现有预设可能偏慢的项

| 预设 | 现有 lifetime | 建议调整 | 原因 |
|------|-------------|----------|------|
| sparkler (仙女棒) | 0.25-0.5s | **保持** ✓ | 已经很好，符合行业标准 |
| snow (雪花) | 4-8s | **保持** ✓ | 氛围类本就需要长时间 |
| explosion (爆炸) | 0.3-0.8s | 收窄为 **0.2-0.5s** | 上限 0.8s 偏慢，爆炸应该更干脆 |

| fireworks (烟花) | 1-2s | **保持** ✓ | 合理范围 |

### 8.3 speed 参数的换算建议

现有 presets 用 px/s 表示速度。如果我们用缓动曲线控制 timing，需要知道：

| 飞行距离 | 推荐时长 | 对应平均速度 (800px画面) | 对应峰值速度(ease-in) |
|----------|----------|------------------------|---------------------|
| 飞出画面(100%) | 0.4s | 2000 px/s | 3500-4000 px/s |
| 飞出画面(150%) | 0.5s | 2400 px/s | 4000-5000 px/s |
| 短距漂移(30%) | 1.0s | 240 px/s | 400-500 px/s |
| 微距飘动(10%) | 2.0s | 40 px/s | 60-80 px/s |

**结论**：现有 explosion 的 speed 400-1000 px/s 在"飞出画面"场景下可能偏慢。如果画面 800px 宽，飞出去需要至少 800px 距离，400px/s 要 2 秒才能飞出 —— 太慢了。建议快速飞出类动画的速度下限提到 **1500-2000 px/s**。

## 九、关键缓动曲线速查表

### CSS cubic-bezier 格式，可直接用于代码

```javascript
const EASING_PRESETS = {
  // --- 基础 ---
  linear:          [0, 0, 1, 1],
  easeInOut:       [0.42, 0, 0.58, 1],        // CSS 默认 ease-in-out

  // --- 入场（减速停稳）---
  easeOut:         [0, 0, 0.58, 1],            // 通用入场
  easeOutCubic:    [0.215, 0.61, 0.355, 1],    // 稍强减速
  easeOutQuart:    [0.165, 0.84, 0.44, 1],     // 更强减速
  easeOutExpo:     [0.19, 1, 0.22, 1],         // 极强减速（快来急停）
  mdDecelerate:    [0, 0, 0, 1],               // Material Design 入场

  // --- 退场（加速离开）---
  easeIn:          [0.42, 0, 1, 1],            // 通用退场
  easeInCubic:     [0.55, 0.055, 0.675, 0.19], // 稍强加速
  easeInQuart:     [0.895, 0.03, 0.685, 0.22], // 更强加速
  easeInExpo:      [0.95, 0.05, 0.795, 0.035], // 极强加速（慢启动→弹射）
```

## 十、tradeoff 与使用建议

### 缓动曲线驱动 vs 物理引擎驱动

| 维度 | 缓动曲线 (Easing) | 物理引擎 (Physics) |
|------|-------------------|-------------------|
| 适用场景 | 确定性动画（A→B） | 不确定动画（爆炸、飘散） |
| 可预测性 | 高：时长固定，终点确定 | 低：受力后轨迹不完全可控 |
| "手感" | 设计感强，精确可控 | 自然感强，每次略有不同 |
| 实现复杂度 | 低 | 中 |
| 我们用在哪 | 字符从原位→目标位的过程 | 字符飞出后的漂移/飘散 |

**建议组合方式**：
1. **退场启动阶段**（0-30%）：用 easing 曲线控制加速，确保"起手"利落
2. **退场飞行阶段**（30-100%）：切换到物理引擎，让 drag/gravity/wind 接管，产生自然的随机轨迹

这正是 C4D MoGraph 的做法：Effector 的强度曲线 = easing 控制启动节奏，Random/Wind = 物理控制后续轨迹。
