# 阶段 1 设计决策：参数推导体系

## 问题：动画的技术参数（速度、节奏、粒子寿命等）怎么确定？

用户不应该被问这些参数，系统应该自己给出正确的值。

## 讨论过的思路

| 思路 | 问题 |
|------|------|
| 让用户选（选择题） | 选择题不该超过 1-2 个，参数收集不是选择题的用途 |
| 全由情绪推导 | 错误——速度/节奏遵循物理事实，不受情绪影响 |
| 由 agent 每次自己猜 | 不稳定，没有基准线 |

## 最终选择：多张映射表，各管各的

参数的来源分为不同维度，每个维度有独立的映射规则：

- **物理特性**（速度、寿命、轨迹、重力）→ 由"这是什么东西"决定
  - 仙女棒火花 = 快速短促（寿命 0.2-0.5s）
  - 雪花 = 缓慢飘落
  - 爆炸 = 瞬间扩散
  - 这些遵循现实世界的物理规律，不受情绪影响

- **视觉风格**（颜色、发光、材质感）→ 由"情绪"决定
  - 温暖浪漫 = 粉金暖色 + 柔和发光
  - 神秘魔法 = 冰蓝紫 + 锐利闪烁
  - 这些是主观审美，由用户的情绪选择驱动

- **更多维度待定**（后续补充）

## 关键洞察

1. **不同参数有不同的"真理来源"。** 物理参数的真理是现实世界，视觉参数的真理是用户情绪。把它们混在一起（"温暖浪漫所以慢一点"）是错误的——仙女棒再浪漫它的火花也是咻咻咻的。

2. **选择题只用来澄清歧义，不用来收集参数。** "仙女棒还是烟花"是歧义，需要选。"速度多快"不是歧义，系统应该知道答案。

3. **映射表不止两张。** 完整的体系会有很多维度（形状、密度、交互方式...），但现在不展开，后续按需补充。
